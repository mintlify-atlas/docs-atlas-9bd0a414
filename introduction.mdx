## What is CBDB Online?

CBDB Online Input System is a web-based platform for managing and contributing to the **China Biographical Database (CBDB)**, a collaborative research database of biographical information about historical Chinese figures from the 7th through 19th centuries.

The input system provides a comprehensive interface for researchers and scholars to:

- **Enter and edit biographical data** of historical Chinese figures
- **Manage complex relationships** including kinship, associations, and social networks
- **Track official appointments** (postings and offices) with temporal and geographical context
- **Document alternate names** and various forms of personal identifiers
- **Record textual references** and scholarly sources
- **Collaborate through proposals** with expert review and approval workflows

<Note>
  The system is built on Laravel 12.x with PHP 8.2+ and uses MariaDB for data persistence. All biographical data follows structured schemas optimized for historical research.
</Note>

## Who Should Use This System?

### Researchers and Scholars

If you study Chinese history, particularly from the 7th through 19th centuries, this system provides:

- A centralized platform for contributing biographical discoveries
- Access to structured data suitable for quantitative analysis
- Tools for tracking relationships and social networks across dynasties

### Data Contributors

Whether you're a graduate student, independent researcher, or crowdsourcing volunteer:

- **Regular users** can directly enter and edit biographical records
- **Crowdsourcing users** submit proposals reviewed by experts
- **Experts** can review, approve, and manage all contributions

### Administrators

System administrators and database managers can:

- Manage user accounts and permissions
- Monitor operations history and audit logs
- Perform batch imports and data maintenance
- Configure code tables and system settings

## Data Structure

The system organizes biographical information into several core categories:

- **Basic Information** (`BIOG_MAIN`): Names, dates, ethnicity, geographic origins
- **Kinship** (`KIN_DATA`): Family relationships following traditional mourning system classifications
- **Offices and Postings** (`POSTING_DATA`, `POSTED_TO_OFFICE_DATA`, `POSTED_TO_ADDR_DATA`): Official appointments with temporal and geographic details
- **Alternate Names** (`ALTNAME_DATA`): Various name forms including style names, courtesy names, posthumous names
- **Associations** (`ASSOC_DATA`): Social relationships including teachers, students, colleagues
- **Entry Paths** (`ENTRY_DATA`): Examination results and paths to official service
- **Texts** (`TEXT_CODES`, `TEXT_INSTANCE_DATA`): Authored works and textual references

<Warning>
  The database uses both single-key and **composite primary key tables**. Composite key tables (like `ALTNAME_DATA`, `POSTED_TO_ADDR_DATA`) require special handling and cannot use standard Eloquent ORM methods.
</Warning>

## Technology Stack

- **Backend**: Laravel 12.x (PHP 8.2+, recommend 8.4)
- **Database**: MariaDB 10.3.39 (production), SQLite (testing)
- **Frontend**: AdminLTE 3 (Bootstrap 4), Vue 3, Vite build system
- **Web Server**: Caddy (production)
- **Authentication**: Laravel Sanctum for API tokens, built-in auth for web
