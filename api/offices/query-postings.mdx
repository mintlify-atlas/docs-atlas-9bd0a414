---
title: 'Query Office Postings'
api: 'POST /api/query_office_postings'
description: 'Query office posting records with complex filtering by office, location, date, and coordinates'
---

Query office posting records (除授記錄) with advanced filtering options including office types, related locations (office and people), time periods, and geographic coordinates.

## Base URL

All API requests should be made to: `https://input.cbdb.fas.harvard.edu`

## Request Parameters

<Note>This endpoint uses **POST** method with `Content-Type: application/json`</Note>

<ParamField body="office" type="array" required>
  Array of office IDs to query
</ParamField>

<ParamField body="useOfficePlace" type="number">
  Enable office location filtering:
  - `1` = Enable (use `officePlace` parameter)
  - `0` = Disable (ignore `officePlace`)
</ParamField>

<ParamField body="officePlace" type="array">
  Array of location IDs related to offices. Only used when `useOfficePlace=1`
</ParamField>

<ParamField body="usePeoplePlace" type="number">
  Enable people location filtering:
  - `1` = Enable (use `peoplePlace` parameter)
  - `0` = Disable (ignore `peoplePlace`)
</ParamField>

<ParamField body="peoplePlace" type="array">
  Array of location IDs related to people. Only used when `usePeoplePlace=1`
</ParamField>

<ParamField body="useDate" type="number">
  Enable date filtering:
  - `1` = Enable (use date-related parameters)
  - `0` = Disable (ignore all date parameters)
</ParamField>

<ParamField body="dateType" type="string">
  Date type for filtering:
  - `"index"` = Use index year (指數年)
  - `"dynasty"` = Use dynasty (朝代)
</ParamField>

<ParamField body="indexStartTime" type="number">
  Start year for index year range. Used when `dateType="index"`
</ParamField>

<ParamField body="indexEndTime" type="number">
  End year for index year range. Used when `dateType="index"`
</ParamField>

<ParamField body="dynStart" type="number">
  Starting dynasty code. Used when `dateType="dynasty"`
</ParamField>

<ParamField body="dynEnd" type="number">
  Ending dynasty code. Used when `dateType="dynasty"`
</ParamField>

<ParamField body="useXy" type="number">
  Include coordinate data in response:
  - `1` = Include coordinates
  - `0` = Exclude coordinates
</ParamField>

<ParamField body="start" type="number">
  Starting record index (0-based)
</ParamField>

<ParamField body="list" type="number">
  Maximum number of records to return
</ParamField>

## Important Notes

<Warning>
  The `use*` parameters (`useOfficePlace`, `usePeoplePlace`, `useDate`) have higher priority than their corresponding data parameters. If a `use*` parameter is set to `0`, the related parameters will be ignored regardless of their values.
</Warning>

<Info>
  When using dynasty filtering (`dateType="dynasty"`), the results will include all records from dynasties within and including `dynStart` and `dynEnd`.
</Info>

## Examples

### Query with people location and index year filtering

<CodeGroup>

```bash curl
curl -X POST "https://input.cbdb.fas.harvard.edu/api/query_office_postings" \
  -H "Content-Type: application/json" \
  -d '{
    "office": [920, 1022, 1023],
    "useOfficePlace": 0,
    "officePlace": [],
    "usePeoplePlace": 1,
    "peoplePlace": [2928, 10522, 12553, 13947, 13949],
    "useDate": 1,
    "dateType": "index",
    "indexStartTime": 960,
    "indexEndTime": 1250,
    "dynStart": null,
    "dynEnd": null,
    "useXy": 1,
    "start": 11,
    "list": 10
  }'
```

```javascript JavaScript
fetch('https://input.cbdb.fas.harvard.edu/api/query_office_postings', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    office: [920, 1022, 1023],
    useOfficePlace: 0,
    officePlace: [],
    usePeoplePlace: 1,
    peoplePlace: [2928, 10522, 12553, 13947, 13949],
    useDate: 1,
    dateType: 'index',
    indexStartTime: 960,
    indexEndTime: 1250,
    dynStart: null,
    dynEnd: null,
    useXy: 1,
    start: 11,
    list: 10
  })
})
.then(response => response.json())
.then(data => console.log(data));
```

```python Python
import requests

response = requests.post(
    'https://input.cbdb.fas.harvard.edu/api/query_office_postings',
    json={
        'office': [920, 1022, 1023],
        'useOfficePlace': 0,
        'officePlace': [],
        'usePeoplePlace': 1,
        'peoplePlace': [2928, 10522, 12553, 13947, 13949],
        'useDate': 1,
        'dateType': 'index',
        'indexStartTime': 960,
        'indexEndTime': 1250,
        'dynStart': None,
        'dynEnd': None,
        'useXy': 1,
        'start': 11,
        'list': 10
    }
)
data = response.json()
```

</CodeGroup>

This query finds all posting records for people who served as Prime Minister (宰相), Left Prime Minister (左丞相), or Right Prime Minister (右丞相) during the Song Dynasty, with people locations in specific areas and index years between 960 and 1250. Returns records 11-20.
